\section{Objectives}

This assignment extends our previous search-based Xiangqi system (Assignment~2) by introducing a \textbf{Machine Learning--based game-playing agent}. The ML agent uses a deep convolutional neural network trained on real-world game data to select moves, replacing the search-based decision process with pure neural inference.

Main objectives:
\begin{itemize}
    \item Implement an ML-based Xiangqi agent using a dual-head CNN (policy + value) inspired by AlphaZero.
    \item Ensure the agent plays according to all Xiangqi rules.
    \item Defeat a random rule-based agent in 10/10 matches regardless of color.
    \item Provide multiple skill levels (Easy, Medium, Hard).
    \item Compare the ML agent with the search-based agents from Assignment~2.
    \item Use only Machine Learning methods for the primary agent (no search fallback).
\end{itemize}

\section{Introduction}

\subsection{Background}

Xiangqi (Chinese Chess) is a deterministic, perfect-information, zero-sum game played on a $10 \times 9$ board with 32 pieces. Its state space ($\sim 10^{48}$) and branching factor ($\sim$30--40) make exhaustive search impractical, motivating the use of learned evaluation and policy functions.

The success of AlphaZero (Silver et al., 2017) demonstrated that deep neural networks can achieve superhuman performance in board games without handcrafted evaluation functions. This project applies similar principles at a practical scale: a ResNet-based dual-head architecture trained on real game records via supervised learning.

\subsection{Relationship to Assignment 2}

Assignment~2 implemented search-based agents (Minimax and Alpha-Beta) with handcrafted evaluation functions. This assignment \textbf{adds} an ML-based agent that operates independently of search. Both agent families share the same core engine (\texttt{core/} module), ensuring identical rule enforcement and state management.

\subsection{System Architecture}

The system retains the modular architecture from Assignment~2 with new ML-specific components:

\section{Data Pipeline and State Representation}

\subsection{Dataset}

The training data comes from the \textbf{Chinese Chess Practical Dataset (CCPD)}, a 427\,MB collection of approximately 120,000 real Xiangqi game records in JSONL format. Each record contains:
\begin{itemize}
    \item Starting FEN position
    \item Sequence of moves with legality annotations
    \item Game result (1--0, 0--1, or 1/2--1/2)
\end{itemize}

Only games with \texttt{all\_moves\_legal = true} are used, ensuring data quality.

\subsection{State Encoding: 15-Channel Tensor}

Each board state is encoded as a $15 \times 10 \times 9$ integer tensor:

\begin{table}[H]
\centering
\caption{15-channel tensor encoding for board state representation}
\begin{tabular}{@{}clP{80mm}@{}}
\toprule
\textbf{Channel} & \textbf{Content} & \textbf{Value} \\
\midrule
0 & Own General position & 1 where piece exists, 0 elsewhere \\
1 & Own Advisors & 1/0 \\
2 & Own Elephants & 1/0 \\
3 & Own Horses & 1/0 \\
4 & Own Rooks & 1/0 \\
5 & Own Cannons & 1/0 \\
6 & Own Soldiers & 1/0 \\
7--13 & Opponent pieces (same 7 types) & 1/0 \\
14 & Side-to-move indicator & 1 if Red, 0 if Black (entire plane) \\
\bottomrule
\end{tabular}
\end{table}

\subsection{Canonical Encoding}

To reduce the pattern space, all positions are normalized to Red's perspective. When it is Black's turn, the board is flipped vertically (row 0 $\leftrightarrow$ row 9) and piece ownership channels are swapped. This ensures the model learns from a single viewpoint, effectively doubling data utilization.

\subsection{Policy Encoding}

Each move is encoded as an index in an 8100-dimensional action space:
\[
\text{index} = \text{src\_row} \times 810 + \text{src\_col} \times 90 + \text{dst\_row} \times 9 + \text{dst\_col}
\]
where $90 \times 90 = 8100$ covers all possible source-destination pairs on the $10 \times 9$ board.

\subsection{Preprocessing Pipeline}

\begin{enumerate}
    \item \textbf{Parse}: Read JSONL records, extract FEN and move sequences.
    \item \textbf{Validate}: Match each move against \texttt{legal\_moves(state)} to confirm legality.
    \item \textbf{Encode}: Convert each (state, move, result) triple into (tensor, policy\_index, value\_target).
    \item \textbf{Augment}: Apply horizontal mirror flip to both state tensor and policy index, doubling the dataset.
    \item \textbf{Quantize}: Store tensors as \texttt{int8} in HDF5 format with chunked compression.
\end{enumerate}

Final dataset sizes:
\begin{itemize}
    \item Training set: 10,208 samples (v2 subset)
    \item Validation set: 2,576 samples
    \item Train/val split ratio: 80/20 (by game index modulo)
\end{itemize}

\section{ML Agent: Network Architecture and Inference}

\subsection{XiangQiResNet Architecture}

The neural network follows a dual-head residual architecture inspired by AlphaZero:

\begin{table}[H]
\centering
\caption{XiangQiResNet architecture specification}
\begin{tabularx}{\textwidth}{@{}lX@{}}
\toprule
\textbf{Component} & \textbf{Specification} \\
\midrule
Input & $15 \times 10 \times 9$ tensor (canonical encoded) \\
Initial convolution & Conv2d($15 \to 128$, $3\times3$, padding=1, no bias) + BatchNorm + ReLU \\
Residual backbone & 5 ResBlocks, each: Conv($3\times3$) + BN + ReLU + Conv($3\times3$) + BN + skip + ReLU \\
Policy head & Conv2d($128 \to 2$, $1\times1$) + BN + ReLU + Flatten + Linear($180 \to 8100$) \\
Value head & Conv2d($128 \to 2$, $1\times1$) + BN + ReLU + Flatten + Linear($180 \to 256$) + Dropout(0.3) + Linear($256 \to 1$) + Tanh \\
Total parameters & $\sim$1,200,000 trainable \\
\bottomrule
\end{tabularx}
\end{table}

\paragraph{Receptive field}
With 5 residual blocks (10 convolutional layers of kernel size $3\times3$), the effective receptive field exceeds $20 \times 20$, surpassing the $10 \times 9$ board dimensions. This allows the network to capture global board patterns from any intersection.

\paragraph{Output interpretation}
\begin{itemize}
    \item \textbf{Policy head}: produces logits over 8100 possible moves; after masking illegal moves and applying softmax, this yields a probability distribution over legal actions.
    \item \textbf{Value head}: produces a scalar in $[-1, 1]$ via Tanh, representing the predicted game outcome from the current player's perspective ($+1$ = win, $-1$ = loss, $0$ = draw).
\end{itemize}

\subsection{Inference Pipeline: 5-Step Move Selection}

The ML Agent selects moves through a sequential 5-step pipeline:

\paragraph{Step 1: Policy inference}
The current game state is encoded as a canonical $15 \times 10 \times 9$ tensor, passed through the CNN in evaluation mode (no gradient), and the 8100-dimensional policy logits are mapped to legal moves only.

\paragraph{Step 2: Capture bonus (reward shaping)}
Capture moves receive score bonuses proportional to captured piece value:

\begin{table}[H]
\centering
\caption{Capture bonus values for reward shaping}
\begin{tabular}{@{}lc@{}}
\toprule
\textbf{Captured Piece} & \textbf{Bonus} \\
\midrule
General & 50.0 \\
Rook & 0.8 \\
Cannon / Horse & 0.5 \\
Advisor / Elephant & 0.1 \\
Soldier & 0.05 \\
\bottomrule
\end{tabular}
\end{table}

This biases the policy toward tactical exchanges without overriding the neural network's strategic judgment.

\paragraph{Step 3: Anti-stalemate terminal check}
For the top 5 scored moves plus all capture moves, the agent simulates the resulting state and checks for terminal conditions:
\begin{itemize}
    \item If a move leads to \textbf{checkmate}: score $+= 1000$ (highest priority).
    \item If a move leads to \textbf{stalemate} (draw): score $= 0$ (eliminated).
\end{itemize}
This prevents the agent from accidentally drawing by capturing all opponent pieces, leaving the opponent's General with no legal moves.

\paragraph{Step 4: Anti-repetition}
Three layers of cycle prevention penalize repetitive moves:
\begin{itemize}
    \item 2-ply reversal: blocks immediate backtracking (A$\to$B then B$\to$A).
    \item 4-ply cycle: detects 4-move loops.
    \item 6-ply cycle: detects 6-move loops.
\end{itemize}
Each violating move receives a 90\% score reduction.

\paragraph{Step 5: Temperature-based selection}
The final move is selected based on difficulty level:
\begin{itemize}
    \item \textbf{Hard} ($\tau = 0$): deterministic greedy selection (argmax).
    \item \textbf{Medium} ($\tau = 0.3$): focused stochastic sampling.
    \item \textbf{Easy} ($\tau = 0.7$): exploratory stochastic sampling.
\end{itemize}

\section{Search Agent (Comparison Baseline)}

To satisfy the assignment requirement of comparing ML-based and search-based agents, we retain the search agents from Assignment~2. This section summarizes their design for completeness.

\subsection{MinimaxAgent}

Exhaustive game-tree search to depth $d$:
\[
V(s) =
\begin{cases}
\max\limits_{a \in A(s)} V(T(s,a)), & \text{if MAX node} \\
\min\limits_{a \in A(s)} V(T(s,a)), & \text{if MIN node}
\end{cases}
\]
Complexity: $O(b^d)$ with $b \approx 35$.

\subsection{AlphaBetaAgent}

Extends Minimax with alpha-beta pruning, transposition table caching, and MVV-LVA move ordering. Best-case complexity: $O(b^{d/2})$.

\subsection{Evaluation Functions}

Two evaluation tiers are available:
\begin{itemize}
    \item \textbf{Basic}: material-only scoring ($\text{eval} = \sum_{\text{own}} V(p) - \sum_{\text{opp}} V(p)$).
    \item \textbf{Advanced}: adds positional bonuses for piece activity (river crossing, file control, palace proximity) and penalties for undeveloped pieces and exposed generals.
\end{itemize}

    \item \textbf{Scalability}: training on more data improves performance without code changes.
\end{enumerate}

\paragraph{Search Agent advantages}
\begin{enumerate}
    \item \textbf{Deterministic}: fully reproducible decisions, easier to debug.
    \item \textbf{No data dependency}: works immediately without dataset or training.
    \item \textbf{Tactical precision}: at depth~4+, search can calculate forced sequences more reliably than 1-ply CNN inference.
\end{enumerate}

\section{Conclusion}

\subsection{Summary of Results}

This project successfully implements a Machine Learning--based Xiangqi agent that satisfies all assignment requirements:

\begin{enumerate}
    \item \textbf{Rule compliance}: both ML and Search agents play according to all Xiangqi rules, verified by 43 automated tests.
    \item \textbf{Defeat random 10/10}: the ML Agent (Hard) achieves a 96.7\% win rate over 30 games (29/30), with two independent 10/10 perfect runs. All wins are by checkmate.
    \item \textbf{Skill levels}: the ML Agent supports 3 levels (Easy/Medium/Hard) via temperature-controlled selection. The Search Agent supports 10 levels via depth and heuristic configuration.
    \item \textbf{ML vs Search comparison}: detailed comparison shows ML is faster ($\sim$0.5\,s vs 5--15\,s) while Search offers deeper tactical calculation at high depth.
    \item \textbf{Pure ML}: the primary agent uses only CNN inference with reward shaping; no Minimax or Alpha-Beta is used as fallback.
\end{enumerate}

\subsection{Technical Contributions}

\begin{itemize}
    \item \textbf{Anti-stalemate mechanism}: a novel terminal-state check on candidate moves prevents the agent from accidentally drawing by over-capturing.
    \item \textbf{Canonical encoding}: normalizing board perspective to Red's viewpoint halves the pattern space and improves data efficiency.
    \item \textbf{Calibrated reward shaping}: capture bonuses tuned to encourage tactical play without causing stalemate.
    \item \textbf{Multi-layer cycle prevention}: three independent repetition detectors (2/4/6-ply) eliminate draw-by-repetition.
\end{itemize}

\subsection{Limitations}

\begin{itemize}
    \item Training used only $\sim$10K samples (v2 subset). Full dataset ($\sim$1M+ samples) would improve generalization.
    \item Training was performed on CPU only ($\sim$160\,s for 5 epochs). GPU training would enable larger models and datasets.
    \item No opening book or endgame tablebase is integrated.
    \item Self-play reinforcement learning (as in AlphaZero) was not implemented.
\end{itemize}

\subsection{Future Work}

\begin{itemize}
    \item Train on the full CCPD dataset with GPU acceleration.
    \item Integrate Monte Carlo Tree Search (MCTS) with the CNN for deeper look-ahead.
    \item Implement self-play training to improve without external data.
    \item Add opening and endgame databases for improved early/late-game play.
    \item Deploy as an online multiplayer application.
\end{itemize}
